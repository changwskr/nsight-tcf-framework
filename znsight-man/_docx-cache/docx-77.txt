F. 오류코드 표준표
F.1 도입 전 안내말
본 부록은 NSIGHT TCF Framework에서 사용하는 오류코드 표준표를 정의한다.
오류코드는 단순한 메시지 번호가 아니다. NSIGHT에서는 오류코드가 표준 응답, 거래로그, 감사로그, 장애추적, 운영자 조치, 재처리 판단, 모니터링 알림의 기준이 된다.
NSIGHT 오류코드 설계 기준은 모든 예외를 TCF/ETF에서 표준 오류코드로 변환하고, OM_ERROR_CODE에서 사용자 메시지와 운영자 메시지를 조회한 뒤, 표준 응답 전문으로 조립하는 구조를 권장한다.
오류 발생
  ↓
Exception 발생
  ↓
TCF / ETF에서 표준 오류코드로 변환
  ↓
OM_ERROR_CODE 조회
  ↓
사용자 메시지 / 운영자 메시지 / 조치 가이드 매핑
  ↓
StandardResponse 조립
  ↓
거래로그 / 감사로그 / 모니터링 기록


F.2 오류코드 기본 형식
오류코드는 다음 형식을 사용한다.
E-{DOMAIN}-{CATEGORY}-{NNNN}

구성요소
의미
표기
예시

Error
고정값
E

오류 발생 영역
대문자
TCF, SV, OM, UD
CATEGORY
오류 분류
대문자
HDR, VAL, BIZ, DB, TIME
NNNN
일련번호
4자리 숫자
0001
예시는 다음과 같다.

E-TCF-SVC-0001
E-TCF-CTL-0001
E-SV-BIZ-0001
E-OM-CODE-0001
E-UD-FILE-0001
E-BT-JOB-0001

오류코드 표준 형식은 E-{DOMAIN}-{CATEGORY}-{NNNN}이며, DOMAIN은 오류 발생 영역, CATEGORY는 오류 분류, NNNN은 일련번호를 의미한다.

F.18 최종 정리
오류코드 표준표의 핵심은 다음이다.
오류코드
= 오류 발생 영역
+ 오류 분류
+ 일련번호

운영 관점에서는 다음 기준이 더 중요하다.
사용자 메시지
= 사용자가 이해할 수 있는 안내

운영자 메시지
= 운영자가 원인을 찾을 수 있는 정보

조치 가이드
= 어디를 확인하고 무엇을 조치할지에 대한 기준

거래로그 / 감사로그
= GUID 기준 장애 추적과 감사 대응

따라서 NSIGHT TCF Framework에서 오류코드는 다음 한 문장으로 정의할 수 있다.
오류코드는 예외를 화면에 보여주기 위한 값이 아니라, 장애를 표준 응답·로그·감사·운영조치로 연결하는 운영 식별자이다.
