Java Method / Field 명명규칙 설계기준
1. 도입 전 안내말
Java Method와 Field 명명규칙은 코드 가독성만을 위한 기준이 아니다.NSIGHT TCF Framework에서는 Method와 Field 이름이 업무 의도, 계층 책임, ServiceId, DTO, Mapper SQL, 거래로그 추적성과 연결되어야 한다.
NSIGHT Java 코딩 스타일 가이드에서도 Method는 lowerCamelCase와 “동사 + 목적어” 구조를 사용하고, Variable은 lowerCamelCase와 의미 있는 명사를 사용하며, Constant는 UPPER_SNAKE_CASE를 사용하도록 정의하고 있다.
핵심 원칙은 다음이다.
Method는 무엇을 하는지 드러내고, Field는 무엇을 담는지 드러내야 한다.

17. 마무리말
Java Method와 Field 명명규칙은 코드 내부의 작은 규칙처럼 보이지만, 실제로는 NSIGHT 운영 추적성과 품질을 좌우한다.
ServiceId
   ↓
Method
   ↓
Field / DTO
   ↓
Mapper Method
   ↓
SQL ID
   ↓
거래로그 / 오류로그

따라서 NSIGHT에서는 다음 기준을 고정한다.
Method는 “동사 + 업무대상”으로 작성하고, Field는 “의미 있는 업무 명사”로 작성한다.
이 기준을 지키면 신규 개발자도 코드를 빠르게 이해할 수 있고, 운영자는 ServiceId, Method, SQL ID, 거래로그를 같은 기준으로 추적할 수 있다.
