21. Header 작성 기준
21.1 도입 전 안내말
NSIGHT TCF Framework에서 Header는 온라인 거래의 공통 통제 영역이다.Body가 업무 데이터를 담는 영역이라면, Header는 이 거래가 어느 업무인지, 어떤 서비스를 실행할 것인지, 누가 요청했는지, 어느 채널·지점에서 요청했는지, 어떻게 추적할 것인지를 담는 영역이다.
NSIGHT 표준 전문은 StandardRequest = header + body, StandardResponse = header + result + body 구조를 기준으로 하며, 기본 Endpoint는 POST /{businessCode}/online, 실행 식별자는 URL이 아니라 header.serviceId이다.
따라서 Header는 다음 기준으로 이해해야 한다.
Header
= 거래 식별 정보
+ 실행 서비스 정보
+ 사용자 / 채널 / 지점 정보
+ 추적 정보
+ 거래통제 / 권한 / 감사 / 로그 기준

21.18 Header Validator 예시
@Component
@RequiredArgsConstructor
public class StandardHeaderValidator {

    private final ServiceCatalogService serviceCatalogService;
    private final ChannelPolicyService channelPolicyService;

    public void validate(StandardHeader header, String pathBusinessCode) {

        require(header.getBusinessCode(), "E-TCF-HDR-0001", "업무코드는 필수입니다.");
        require(header.getServiceId(), "E-TCF-HDR-0002", "ServiceId는 필수입니다.");
        require(header.getTransactionCode(), "E-TCF-HDR-0003", "거래코드는 필수입니다.");
        require(header.getProcessingType(), "E-TCF-HDR-0004", "처리유형은 필수입니다.");
        require(header.getChannelId(), "E-TCF-HDR-0005", "채널ID는 필수입니다.");
        require(header.getUserId(), "E-TCF-HDR-0006", "사용자ID는 필수입니다.");
        require(header.getBranchId(), "E-TCF-HDR-0007", "지점코드는 필수입니다.");

        if (!pathBusinessCode.equalsIgnoreCase(header.getBusinessCode())) {
            throw new BusinessException(
                    "E-TCF-HDR-0011",
                    "URL 업무코드와 Header 업무코드가 일치하지 않습니다."
            );
        }

        if (!header.getServiceId().startsWith(header.getBusinessCode() + ".")) {
            throw new BusinessException(
                    "E-TCF-HDR-0012",
                    "업무코드와 ServiceId가 일치하지 않습니다."
            );
        }

        if (!header.getTransactionCode().startsWith(header.getBusinessCode() + "-")) {
            throw new BusinessException(
                    "E-TCF-HDR-0013",
                    "업무코드와 거래코드가 일치하지 않습니다."
            );
        }

        serviceCatalogService.validateActiveService(
                header.getServiceId(),
                header.getTransactionCode()
        );

        channelPolicyService.validateChannel(header.getChannelId());
    }

    private void require(String value, String errorCode, String message) {
        if (value == null || value.isBlank()) {
            throw new BusinessException(errorCode, message);
        }
    }
}

21.25 마무리말
Header는 NSIGHT 표준 전문에서 가장 중요한 공통 영역이다.업무 개발자가 Body만 잘 작성해도 Header가 부정확하면 TCF Framework는 해당 거래를 안전하게 실행할 수 없다.
Header가 정확해야 다음이 가능하다.

21.26 시사점
NSIGHT 개발에서 Header를 잘못 작성하면 단순 입력 오류가 아니라 거래통제 우회, 권한 검증 실패, 로그 추적 불가, 감사 누락, 장애 분석 불가로 이어진다.
따라서 개발자는 다음 원칙을 지켜야 한다.
Header는 업무 데이터가 아니라 거래 실행을 통제하는 공통 계약이다.Header 값은 클라이언트가 보낸 그대로 믿지 말고, STF에서 검증·보정해야 한다.ServiceId, 거래코드, 업무코드, 사용자, 채널, 지점은 반드시 하나의 정합성 있는 조합으로 관리해야 한다.
