55. 통합 테스트 기준
55.1 도입 전 안내말
NSIGHT TCF Framework에서 통합 테스트는 단순히 API가 호출되는지 확인하는 테스트가 아니다.통합 테스트는 화면 또는 호출자가 POST /{businessCode}/online으로 표준 전문을 보내고, TCF 공통 처리와 업무 처리, DB 처리, 거래로그, 오류 응답까지 End-to-End로 정상 동작하는지 검증하는 절차이다.
NSIGHT 전처리·후처리 기준에서도 모든 온라인 거래는 동일한 방식으로 수신, 검증, 추적, 처리, 응답되어야 하며, GUID, TraceId, 인증, 권한, 세션, 마스킹, 감사, Timeout, 재처리, 모니터링을 공통 처리 대상으로 정의하고 있다.

55.11 RestAssured 기반 통합 테스트 예시
@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)
@ActiveProfiles("stg")
class SvCustomerIntegrationTest {

    @LocalServerPort
    int port;

    @Test
    void 고객요약조회_통합테스트_정상() {
        Map<String, Object> request = Map.of(
            "header", Map.of(
                "guid", "G202607050001",
                "traceId", "T202607050001",
                "serviceId", "SV.Customer.selectSummary",
                "transactionCode", "SV-INQ-0001",
                "businessCode", "SV",
                "serviceName", "고객요약조회",
                "userId", "user001",
                "channelId", "WEBTOP",
                "branchId", "123456"
            ),
            "body", Map.of(
                "customerNo", "1234567890"
            )
        );

        given()
            .port(port)
            .contentType("application/json")
            .cookie("JSESSIONID", "TEST_SESSION_ID")
            .body(request)
        .when()
            .post("/sv/online")
        .then()
            .statusCode(200)
            .body("header.serviceId", equalTo("SV.Customer.selectSummary"))
            .body("header.businessCode", equalTo("SV"))
            .body("result.resultCode", equalTo("COM-0000"))
            .body("body.customerNo", equalTo("1234567890"));
    }
}


55.12 MockMvc 기반 통합 테스트 예시
@SpringBootTest
@AutoConfigureMockMvc
@ActiveProfiles("dev")
class OmUserIntegrationTest {

    @Autowired
    private MockMvc mockMvc;

    @Test
    void 사용자조회_통합테스트_정상() throws Exception {
        String requestJson = """
        {
          "header": {
            "guid": "G202607050002",
            "traceId": "T202607050002",
            "serviceId": "OM.User.inquiry",
            "transactionCode": "OM-INQ-0001",
            "businessCode": "OM",
            "serviceName": "사용자조회",
            "userId": "admin",
            "channelId": "WEBTOP",
            "branchId": "000000"
          },
          "body": {
            "userName": "홍"
          }
        }
        """;

        mockMvc.perform(post("/om/online")
                .contentType(MediaType.APPLICATION_JSON)
                .content(requestJson)
                .cookie(new Cookie("JSESSIONID", "TEST_SESSION_ID")))
            .andExpect(status().isOk())
            .andExpect(jsonPath("$.header.serviceId").value("OM.User.inquiry"))
            .andExpect(jsonPath("$.result.resultCode").value("COM-0000"));
    }
}

55.21 마무리말
NSIGHT 통합 테스트의 핵심은 업무 프로그램 하나가 정상 동작하는지가 아니라, TCF 공통 파이프라인과 업무 계층, DB, 로그, 보안, Timeout이 하나의 거래로 연결되어 정상 동작하는지 증명하는 것이다.
통합 테스트
= HTTP 표준 전문 호출
+ TCF 전처리 검증
+ ServiceId Dispatcher 검증
+ 업무 Handler 실행 검증
+ DB/Mapper 검증
+ ETF 표준 응답 검증
+ 거래로그/감사로그 검증
+ Timeout/권한/마스킹 검증

55.22 시사점
단위 테스트가 코드 내부의 책임을 검증한다면, 통합 테스트는 NSIGHT 거래 실행 구조 전체가 운영 가능한지를 검증한다.
따라서 NSIGHT 개발가이드의 통합 테스트 기준은 다음 한 문장으로 정리할 수 있다.
요청 전문 한 건이 들어와서 응답 전문 한 건으로 끝날 때까지, Header·세션·권한·거래통제·업무처리·DB·로그·오류코드가 모두 연결되어야 한다.
