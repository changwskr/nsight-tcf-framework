# 화면번호 명명규칙

> **NSIGHT TCF 명명규칙 상세** · 원본: [`znsight-guide`](../znsight-guide/) · 갱신: 2026-07-05

화면번호 명명규칙 설계기준
## 1. 도입 전 안내말

화면번호는 단순히 화면을 구분하기 위한 번호가 아니다.NSIGHT TCF에서는 화면번호가 메뉴권한, 기능권한, ServiceId, 거래코드, 거래로그, 감사로그, 장애추적과 연결된다.
NSIGHT 표준 전문 Header에도 menuId, screenId가 포함되어 있으며, serviceId, transactionCode, businessCode, guid, traceId와 함께 거래 식별과 추적에 사용된다. 또한 TCF 구조에서는 모든 온라인 거래가 Header 기준으로 검증·세션·권한·거래통제·Timeout·로그·감사를 수행한 뒤 ServiceId에 매핑된 Handler를 실행한다.
따라서 화면번호는 다음 흐름과 반드시 연결되어야 한다.
업무코드
```text
  ↓
MenuId
  ↓
ScreenId
  ↓
ServiceId
↓
TransactionCode
↓
기능권한
↓

```

## 2. 화면번호 명명 최상위 원칙

| 원칙 | 설계 기준 |
| --- | --- |
| 업무코드 우선 | 화면번호는 반드시 업무코드로 시작한다 |
| 화면유형 포함 | 메인, 서브, 팝업, 탭, 위저드, 리포트 등 화면 유형을 포함한다 |
| 일련번호 고정 | 업무별·유형별로 4자리 일련번호를 사용한다 |
| MenuId와 분리 | 메뉴번호와 화면번호는 다르다. 메뉴는 네비게이션, 화면은 실제 UI 단위다 |
| ServiceId와 연결 | 한 화면에서 호출하는 ServiceId 목록을 관리한다 |
| 권한관리 연계 | 메뉴권한과 기능권한의 기준으로 사용한다 |
| 로그 추적 가능 | 거래로그에 screenId를 남겨 어느 화면에서 발생한 거래인지 추적한다 |
| 재사용 금지 | 폐기된 화면번호는 재사용하지 않는다 |
| 의미 변경 금지 | 기존 화면번호의 업무 의미를 바꾸지 않는다 |
| 임시번호 금지 | TEST001, SCREEN1, POPUP1 같은 임시 번호 사용 금지 |

## 3. 화면번호 표준 형식

NSIGHT 화면번호는 다음 형식을 권장한다.
{업무코드}{화면유형코드}{NNNN}

예시:
SVMAIN0001
SVSUB0001
SVPOP0001
OMDASH0001
OMMENU0001
CMWIZ0001

| 구성요소 | 자리수 | 설명 | 예시 |
| --- | --- | --- | --- |
| 업무코드 | 2자리 | NSIGHT 업무코드 | SV, CM, OM |
| 화면유형코드 | 3~5자리 | 화면 성격 | MAIN, SUB, POP, DASH |
| 일련번호 | 4자리 | 업무·유형별 순번 | 0001 |

하이픈을 사용하는 방식도 가능하지만, 화면번호는 Header, 로그, DB 컬럼, 권한 테이블에서 자주 사용되므로 운영 표준값은 하이픈 없는 고정 문자열을 권장한다.

## 5. 화면번호와 메뉴번호의 관계

화면번호와 메뉴번호는 분리한다.
MenuId   = 사용자가 메뉴 트리에서 클릭하는 항목
ScreenId = 실제 표시되는 화면 단위

| 구분 | 역할 | 예시 |
| --- | --- | --- |
| MenuId | 메뉴 트리, 권한그룹별 메뉴 접근 기준 | OMMENU0001 |
| ScreenId | 실제 UI 화면 식별 | OMDASH0001 |
| ServiceId | 화면에서 호출하는 업무 서비스 | OM.Dashboard.inquiry |
| FunctionId | 화면의 버튼·기능 권한 | SEARCH, SAVE, DELETE, DOWNLOAD |
| 예시: | 메뉴명 | MenuId |
| ScreenId | 대표 ServiceId | 운영 대시보드 |
| OMMENU0001 | OMDASH0001 | OM.Dashboard.inquiry |
| 거래로그 조회 | OMMENU0002 | OMLIST0001 |
| OM.TxLog.selectList | 사용자 관리 | OMMENU0003 |
| OMLIST0002 | OM.User.selectList | 사용자 상세 |
| OMMENU0003 | OMDTL0001 | OM.User.selectDetail |
| 캠페인 등록 | CMMENU0001 | CMREG0001 |

CM.Campaign.create
운영 대시보드 설계에서도 화면은 /ui/om/admin/dashboard.html, Backend는 tcf-om, ServiceId는 OM.Dashboard.inquiry로 연결된다. 이처럼 화면, Backend, ServiceId가 함께 관리되어야 한다.

## 16. 마무리말

화면번호 명명규칙의 핵심은 다음과 같다.
화면번호는 UI 식별자가 아니라
메뉴권한, 기능권한, ServiceId, 거래로그, 감사로그를 연결하는
운영 추적 식별자다.

따라서 NSIGHT 화면번호는 {업무코드}{화면유형코드}{4자리일련번호} 형식으로 관리한다.예를 들어 SVLIST0001은 SV 업무의 목록화면이고, OMDASH0001은 OM 운영 대시보드이며, CMWIZ0001은 캠페인 위저드 화면이다. 이렇게 정리하면 운영자는 거래로그의 screenId만 보고도 어느 업무의 어떤 화면에서 어떤 거래가 발생했는지 즉시 추적할 수 있다.
