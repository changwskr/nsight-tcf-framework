## 70. FAQ / Troubleshooting

### 70.1 도입 전 안내말

FAQ / Troubleshooting은 개발자가 NSIGHT TCF Framework를 개발·빌드·배포·운영 중 자주 만나는 문제를 증상 → 원인 → 확인 방법 → 조치 방법 순서로 빠르게 판단할 수 있도록 정리한 장이다.
NSIGHT TCF Framework는 HTTP/JSON 표준 전문 + ServiceId Dispatcher + STF/TCF/ETF 공통 처리 파이프라인 구조를 기준으로 하며, 모든 온라인 거래는 /{businessCode}/online으로 수신되고 header.serviceId를 기준으로 Handler가 실행된다. 기존 설계 기준에서도 TCF는 Header 검증, 세션·권한, 거래통제, Timeout, 로그, 응답 조립을 공통 파이프라인에서 처리하는 구조로 정의되어 있다.
따라서 장애나 오류가 발생하면 무작정 소스를 수정하기보다 다음 순서로 확인한다.
## 1. 어떤 환경인가?

   local / dev / stg / prod

## 2. 어떤 모듈인가?

   tcf-core / tcf-web / sv-service / tcf-om / tcf-gateway / tcf-jwt

## 3. 어떤 단계에서 실패했는가?

   build / bootRun / WAR / deploy / runtime / transaction

## 4. 어떤 거래인가?

   businessCode / serviceId / transactionCode

## 5. 어떤 추적값이 있는가?

   GUID / TraceId / TransactionId

## 6. 어느 계층에서 실패했는가?

   STF / Dispatcher / Handler / Service / DAO / Mapper / ETF

### 70.11 MyBatis / SQL FAQ

Q17. Invalid bound statement 오류가 발생한다
| 원인 | 설명 |
| --- | --- |
| --- | --- |
| Mapper XML namespace 불일치 | Interface FQCN과 다름 |
| SQL id 불일치 | Interface Method명과 다름 |
| --- | --- |
| Mapper XML 미포함 | WAR에 XML 누락 |
| Mapper Scan 누락 | Package Scan 범위 오류 |
| --- | --- |
| 파일 경로 오류 | resources/mapper/** 위치 불일치 |
| 확인: | jar tf sv.war \| grep mapper |

정상 기준:
Interface:
com.nh.nsight.sv.persistence.mapper.SvCustomerMapper.selectSummary

XML:
<mapper namespace="com.nh.nsight.sv.persistence.mapper.SvCustomerMapper">
  <select id="selectSummary">

MyBatis는 DAO/Mapper 계층에 위치하며 SQL 실행 책임만 갖고, Mapper 표준과 Query Timeout, SQL ID 관측성을 통제해야 한다.
