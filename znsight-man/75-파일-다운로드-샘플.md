## 이원 구조 (18.2)

| 계층 | API | 내용 |
|------|-----|------|
| **메타·권한** | `POST /om/online` `OM.File.*` | TCF 표준 전문 |
| **Binary** | `POST /ud/files/upload` `GET /ud/files/{id}/download` | Multipart/Stream |

JSON Body에 **대용량 Binary ❌**

## 전체 (18.3)

```
목록/메타: OM.File.inquiry → OM_FILE_META
업로드: /ud/files/upload → Storage + 메타 + 감사
다운로드: /ud/files/{id}/download → 권한 → Stream + 이력
```

Apache: `/ud/*` Reverse Proxy 포함

## 업로드 (18.5)

1. 세션/JWT·권한  
2. 크기·확장자·MIME  
3. (선택) 바이러스 검사  
4. Storage 저장  
5. 메타 INSERT + OM_AUDIT_LOG  

## 다운로드

- fileId 권한  
- UD_FILE_DOWNLOAD_LOG  
- Watermark/DRM (선택)  

## Handler

- `OmFileDownloadHandler` — OM.File.*, UD 연계  
- 메타: Facade → Service → DAO (6계층)

## 보관·삭제

Batch Job — 보관기간 만료 → Storage+메타 삭제 (17장)
