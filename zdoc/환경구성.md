# TCF 환경 구성 정리

NSIGHT TCF의 **환경 구성**은 **프로파일(local/dev/prod)**, **배포 모드(bootRun/ztomcat)**, **설정 3계층(OS·JVM·Spring yml)** 으로 나뉩니다.

| 관점 | 역할 | 핵심 |
|------|------|------|
| **프로파일** | local / dev / prod | Spring `spring.profiles.active` |
| **배포 모드** | bootRun vs ztomcat | 포트 분리 vs 8080 게이트웨이 |
| **설정 SoT** | tcf-cicd | yml·setenv·Apache 동기화 |
| **공유 DB** | H2 file | `nsight.txlog.path` → `data/nsight-txlog` |
| **환경변수** | 운영 오버라이드 | `NSIGHT_GATEWAY_BASE_URL` 등 |

---

## 1. 개요 — 환경 3단계

| 프로파일 | 용도 | 전형적 실행 |
|----------|------|-------------|
| **`local`** | 개발자 PC, bootRun | `gradle :sv-service:bootRun` |
| **`dev`** | 통합·검증 (ztomcat) | `ztomcat/conf/setenv.*` → `-Dspring.profiles.active=dev` |
| **`prod`** | 운영 Tomcat | `-Dspring.profiles.active=prod` + 환경변수 |

**`prod` 프로파일 그룹:** `prod` 활성 시 `dev` yml도 함께 로드 (`tcf-web` `spring.profiles.group.prod`).

```yaml
spring:
  profiles:
    group:
      prod:
        - dev
```

운영 전용 값(URL, DB)은 `application-prod.yml`에서 **오버라이드**만 담습니다.

---

## 2. 설정 3계층 (우선순위)

```text
JVM -D (최우선)
  > OS 환경변수
    > application-{profile}.yml
      > application.yml
        > AutoConfiguration 기본값
```

| 계층 | 예 | 설정 위치 |
|------|-----|-----------|
| **OS 환경변수** | `NSIGHT_TXLOG_PATH`, `NSIGHT_GATEWAY_BASE_URL` | shell / Windows |
| **JVM 시스템 속성** | `-Dnsight.txlog.path=...`, `-Dspring.profiles.active=dev` | bootRun, `setenv.*`, `CATALINA_OPTS` |
| **Spring placeholder** | `${NSIGHT_SV_DB_URL:...}` | `application-*.yml` |

Spring Boot 바인딩:

```text
NSIGHT_TXLOG_PATH  ↔  nsight.txlog.path
SPRING_PROFILES_ACTIVE  ↔  spring.profiles.active
```

상세: [docs/manual/environment-variables.md](../docs/manual/environment-variables.md)

## 3. 배포 모드 — bootRun vs ztomcat

### 3.1 bootRun (local)

모듈별 **독립 JVM**, **개별 포트**.

```text
tcf-ui (:8099) ──Relay──► tcf-om (:8097) / sv-service (:8086) / …
tcf-uj (:8102) ──Relay──► tcf-gateway (:8100) ──► 업무 WAS
tcf-ui/uJ ──Bearer──► tcf-gateway (:8100) ──► tcf-om (OM JWT)
tcf-jwt (:8110) — JWKS 발급
tcf-batch (:8098) ──수집──► 각 모듈 Actuator (127.0.0.1:port)
```

루트 `build.gradle`이 모든 `bootRun`에 공통 적용:

| systemProperty | 값 | 목적 |
|----------------|-----|------|
| `workingDir` | 프로젝트 루트 | H2·data 경로 통일 |
| `spring.profiles.active` | `local` | 프로파일 |
| `nsight.txlog.path` | `{root}/data/nsight-txlog` | 거래로그·SESSIONDB 공유 |
| `file.encoding` | UTF-8 | 한글·전문 |

```bash
gradle :sv-service:bootRun
gradle :tcf-om:bootRun
gradle :tcf-batch:bootRun
gradle :tcf-ui:bootRun
```

### 3.2 ztomcat (dev)

**Tomcat 8080** 단일 게이트웨이, **15+ WAR** context (업무 9 + om + batch + ui + uj + gw + jwt + …).

```text
http://localhost:8080/ui     → tcf-ui
http://localhost:8080/uj     → tcf-uj
http://localhost:8080/gw     → tcf-gateway
http://localhost:8080/jwt    → tcf-jwt
http://localhost:8080/om     → tcf-om
http://localhost:8080/batch  → tcf-batch
http://localhost:8080/sv     → sv-service
… (업무 WAR)
```

| 항목 | 값 |
|------|-----|
| Tomcat | 10.1.34 (Jakarta EE 10) |
| JDK | **21 필수** |
| 프로파일 | **`dev`** (`setenv.*`) |
| H2 TCP | 9092 (공유 `nsight_om`) |

```bat
cd ztomcat
h2-txlog.ps1 start
deploy-wars.bat all
start.bat
```

상세: [ztomcat/README.md](../ztomcat/README.md)

### 3.3 UI deployment-mode (Spring 프로파일과 별개)

| Spring | `nsight.tcf-ui.deployment-mode` | Relay 대상 |
|--------|--------------------------------|------------|
| `local` | `bootrun` | `http://127.0.0.1:{port}` |
| `dev` / `prod` | `tomcat` | `${nsight.gateway.base-url}/{ctx}` |

## 5. 공유 데이터 경로

### 5.1 `nsight.txlog.path`

거래로그 H2·OM 운영 DB·Spring Session **공유 파일 DB** 루트.

| 기본 | `{프로젝트루트}/data/nsight-txlog` |
|------|-------------------------------------|
| 파일 | `nsight_om.mv.db` (H2 file) |
| bootRun | Gradle `bootRun` 자동 `-D` |
| ztomcat | `NSIGHT_TXLOG_PATH` → `setenv.*` |
| 자동 해석 | `NsightTxlogPathEnvironmentPostProcessor` (settings.gradle 상위 탐색) |

**용도:**

- `TCF_TX_LOG` — 거래로그 (별도 DS, file H2)
- `SPRING_SESSION` — OM·Gateway SESSIONDB
- `tcf-om` Primary DB — `h2:file:.../nsight_om`
- `tcf-batch` — JDBC 집계·`OM_*_STATUS`

상세: [세션관리.md](세션관리.md), [온라인처리.md](온라인처리.md)

### 5.2 업무 WAR 업무 DB (local)

| 구분 | URL 예 |
|------|--------|
| 업무 Primary | `jdbc:h2:mem:nsight_{code}` (모듈별 in-memory) |
| 거래로그 | `tcf-web` 기본 → file H2 (`separate: true`) |

`tcf-om`만 Primary = file H2 `nsight_om` (세션·OM 테이블).

### local (bootRun)

| 항목 | URL |
|------|-----|

| OM | `http://127.0.0.1:8097/om/online` |

| OM → batch | `nsight.om.batch-service-url: http://127.0.0.1:8098/batch` |

### dev (ztomcat)

| 항목 | URL |
|------|-----|
| 게이트웨이 | `http://127.0.0.1:8080` |
| OM | `http://127.0.0.1:8080/om` |
| Relay | `http://127.0.0.1:8080/ui/api/relay/OM/online` |

### prod (운영)

| 환경변수 | 용도 |
|----------|------|
| `NSIGHT_GATEWAY_BASE_URL` | 공개 API·Relay (Apache 443 등) |
| `NSIGHT_{CODE}_DB_URL` | 업무·OM Oracle |
| `NSIGHT_TXLOG_JDBC_URL` | 거래로그 DB (운영 DB) |
| `NSIGHT_BATCH_SERVICE_URL` | OM → batch |
| `NSIGHT_UPDOWNLOAD_PATH` | OM 파일 저장 |

## 7. 설정 파일 구조

모듈별 **`application.yml` + `application-{local,dev,prod}.yml`**.

| 모듈 | local | dev | prod | 비고 |
|------|-------|-----|------|------|
| `tcf-web` | ✅ | ✅ | ✅ | 프레임워크 JAR, AutoConfiguration |
| `tcf-core` | ✅ | ✅ | ✅ | `TcfProperties` |
| `tcf-om` | ✅ | ✅ | ✅ | Session JDBC, Cache |
| `tcf-batch` | ✅ (`application-local.yml`) | ✅ | ✅ | 수집 target URL |
| `tcf-ui` | ✅ | ✅ | ✅ | Relay, deployment-mode |
| `*-service` | ✅ | ✅ | ✅ | 포트·mem DB명만 상이 |

**SoT (Source of Truth):** [`tcf-cicd/`](../tcf-cicd/README.md)

```text
tcf-cicd/
├── local/ dev/ prod/
│   ├── spring/{module}/application-{profile}.yml
│   ├── ztomcat/setenv.*
│   └── env/.env.example
└── scripts/sync-to-framework.ps1   # 빌드 전 framework 반영
```

워크플로:

```powershell
# tcf-cicd 수정 후 framework에 반영
cd tcf-cicd/scripts
.\sync-to-framework.ps1 -Profile dev
```

## 9. 모듈별 환경 차이 (요약)

| 모듈 | Spring Boot | TCF | DB | 특이 설정 |
|------|-------------|-----|-----|-----------|
| `*-service` | ● WAR | ● Handler | H2 mem + txlog file | `session.store-type: none` |
| `tcf-om` | ● WAR | ● | H2 file `nsight_om` | Spring Session JDBC, Cache |
| `tcf-batch` | ● WAR | ✕ | JDBC → nsight_om | `transaction-log-enabled: false` |
| `tcf-ui` | ● WAR | ✕ | 없음 | Relay only |
| `tcf-gateway` | ● WAR | ✕ | SESSIONDB | user-session-sync |
| `tcf-core/web` | ✕ JAR | AutoConfig | — | `TcfProperties`, DS |

### 업무 WAR 공통 (`application.yml`)

| 영역 | typical |
|------|---------|
| `spring.profiles.default` | `local` |
| `server.port` | 모듈별 |
| `spring.datasource` | H2 mem, `auto-commit: false` |
| `spring.transaction.default-timeout` | 5s |
| `mybatis.default-statement-timeout` | 3s |
| `nsight.tcf.session-validation-enabled` | `false` |
| `nsight.tcf.transaction-log-enabled` | `true` |

### tcf-om 전용

| 설정 | 값 |
|------|-----|
| `spring.session.store-type` | `jdbc` |
| `nsight.tcf.idempotency-enabled` | `false` |
| `transaction-log-datasource.separate` | `false` (Primary 동일) |
| `nsight.om.batch-service-url` | 환경별 batch URL |

## 13. 환경 전환 체크리스트

### local 개발

- [ ] JDK 21, Gradle 8.x
- [ ] `gradle :sv-service:bootRun` — 로그 `profile is active: "local"`
- [ ] `data/nsight-txlog` 생성·H2 file 공유 확인
- [ ] UI `:8099` Relay → OM `:8097` 동작

### dev (ztomcat)

- [ ] `setenv` → `spring.profiles.active=dev`
- [ ] `h2-txlog.ps1 start` (공유 H2)
- [ ] `deploy-wars all` + `start.bat`
- [ ] `http://localhost:8080/ui/om/admin/login.html`

### prod

- [ ] `NSIGHT_GATEWAY_BASE_URL` 설정
- [ ] `spring.profiles.active=prod` (그룹으로 dev 포함)
- [ ] DB URL 환경변수 (`NSIGHT_*_DB_URL`)
- [ ] Apache reverse proxy ([docs/architecture/23-env-apache.md](../docs/architecture/23-env-apache.md))

## 14. 트러블슈팅 (흔한 이슈)

| 증상 | 원인 | 조치 |
|------|------|------|
| ztomcat `/sv/online` 404 | JDK ≠ 21 | Tomcat JDK 21 확인 |
| 세션·로그인 불일치 | txlog path 불일치 | `NSIGHT_TXLOG_PATH` / `-Dnsight.txlog.path` 통일 |
| Relay 실패 (local) | deployment-mode | `bootrun` + 개별 포트 |
| Relay 실패 (ztomcat) | deployment-mode | `tomcat` + gateway URL |
| batch 수집 전부 FAIL | startup delay 부족 | `initial-delay-ms` 증가 |
| 한글 깨짐 | encoding | `-Dfile.encoding=UTF-8` |
