# NSIGHT TCF 솔루션 환경 구성 (Spring Boot 전체)

NSIGHT TCF 솔루션의 **Spring Boot 3.3.5 + Java 21** 환경을 **기동·프로파일·설정·AutoConfiguration·모듈별 yml** 관점에서 한 문서에 정리합니다.

| 관점 | 핵심 |
|------|------|
| **기술 스택** | Spring Boot 3.3.5, Java 21, MyBatis 3.0.3, H2(로컬) |
| **실행 형태** | bootRun(Embedded Tomcat) / WAR(외부 Tomcat 10.1) |
| **프로파일** | `local` · `dev` · `prod` (`prod` → `dev` 그룹 포함) |
| **설정 SoT** | `tcf-cicd/` → `sync-to-framework.ps1` |
| **프레임워크 JAR** | `tcf-core` + `tcf-web`(AutoConfiguration) + `tcf-cache`(선택) |

배포 토폴·포트·ztomcat: [환경구성.md](환경구성.md)

## 2. 모듈 분류 — Spring Boot 관점

```text
[라이브러리 — main() 없음, AutoConfiguration/Component]
  tcf-util          Spring 비의존
  tcf-core          @Component (TCF/STF/ETF), EnvironmentPostProcessor
  tcf-web           AutoConfiguration.imports (DS, MyBatis, 거래로그, 거래통제, Timeout)
  tcf-cache         AutoConfiguration.imports (EhCache)

[실행 WAR — @SpringBootApplication]
  *-service ×9     업무 (IC~MG) + Handler/Facade
  tcf-om           OM Admin, Spring Session JDBC, Cache
  tcf-batch        수집 스케줄 (MyBatis 없음)
  tcf-ui           Relay UI (TCF JAR 없음)
  tcf-gateway      API Gateway, SESSIONDB 관문
  tcf-uj           Gateway 경유 UI
  tcf-jwt          JWT 발급·JWKS (검증=tcf-gateway)

[비-Spring]
  tcf-cicd, tcf-scripts, ztomcat
```

**의존 방향:** `tcf-util → tcf-core → tcf-web → (tcf-cache) → 실행 WAR`

## 3. 기동 방식

### 3.1 bootRun (local)

```bash
gradle :sv-service:bootRun
gradle :tcf-om:bootRun
```

```text
NsightXxApplication.main()
  → SpringApplication.run()
  → Embedded Tomcat
  → 단일 JVM · 모듈별 포트
```

**루트 `build.gradle` — 모든 bootRun 공통:**

| systemProperty / 설정 | 값 |
|----------------------|-----|
| `workingDir` | 프로젝트 루트 |
| `spring.profiles.active` | **`local`** |
| `nsight.txlog.path` | `{root}/data/nsight-txlog` |
| `file.encoding` | UTF-8 |

### 3.2 WAR + 외부 Tomcat (dev/prod)

```text
Tomcat start → WAR deploy (/sv, /om, …)
  → NsightWarBootstrap.configure(SpringApplicationBuilder)
  → ApplicationContext (Embedded Tomcat 없음)
  → spring.profiles.active=dev|prod (setenv)
```

```java
// tcf-web — 업무·tcf-om·tcf-batch
public abstract class NsightWarBootstrap extends SpringBootServletInitializer {
    protected SpringApplicationBuilder configure(SpringApplicationBuilder app) {
        return app.sources(source);
    }
}
```

| | bootRun | WAR (ztomcat) |
|---|---------|---------------|
| Tomcat | Embedded | 외부 10.1 |
| 프로파일 | `local` (Gradle) | `dev` / `prod` |
| 포트 | 8082~8110 | **8080** 게이트웨이 |
| starter-tomcat | runtime | `providedRuntime` |

## 4. Spring Boot 기동 타임라인

```text
Phase 0  JVM start — main() 또는 Tomcat WAR loader
Phase 1  Environment (Context 이전)
           · NsightTxlogPathEnvironmentPostProcessor
           · application.yml + application-{profile}.yml
Phase 2  ApplicationContext refresh
           · @SpringBootApplication / @ComponentScan("com.nh.nsight")
           · AutoConfiguration.imports (tcf-web, tcf-cache)
           · @MapperScan, @Configuration (@EnableJdbcHttpSession 등)
Phase 3  Bean 생성
           · DataSource → SqlSessionFactory → Mapper
           · TransactionDispatcher → Handler Map
           · TCF / STF / ETF / OnlineTransactionController
Phase 4  초기화 콜백
           · @PostConstruct (TransactionLogSchemaInitializer 등)
           · spring.sql.init (tcf-om schema.sql/data.sql)
           · ApplicationRunner (@Order — OmDatabaseMigration, DashboardCollectStartupRunner)
Phase 5  Web Ready
           · DispatcherServlet, Filter (GuidMdcCleanupFilter)
           · Actuator endpoints
Phase 6  HTTP 요청 → TCF.process()
```

## 5. 프로파일 (local / dev / prod)

| 프로파일 | 실행 | 효과 |
|----------|------|------|
| **`local`** | bootRun | 개별 포트, file/mem H2, UI `deployment-mode: bootrun` |
| **`dev`** | ztomcat setenv | 8080 context, H2 TCP 9092, file logging, **13 WAR** 배포·Actuator 수집 |
| **`prod`** | 운영 Tomcat | `dev` 그룹 + 운영 URL/Oracle env |

```yaml
# tcf-web.jar
spring:
  profiles:
    default: local
    group:
      prod:
        - dev
```

| 활성화 | 위치 |
|--------|------|
| bootRun | Gradle `-Dspring.profiles.active=local` |
| ztomcat | `ztomcat/conf/setenv.*` → `dev` |
| 운영 | `setenv.prod.*` → `prod` |
| 오버라이드 | `SPRING_PROFILES_ACTIVE`, `-Dspring.profiles.active` |

## 6. 설정 로딩 우선순위

```text
(낮음) tcf-web.jar / tcf-cache.jar application.yml
     → {실행모듈}/application.yml
     → application-{profile}.yml
     → OS 환경변수
     → JVM -D
(높음) EnvironmentPostProcessor addFirst (nsight.txlog.path)
```

**바인딩 규칙:**

```text
NSIGHT_TXLOG_PATH           ↔  nsight.txlog.path
SPRING_PROFILES_ACTIVE      ↔  spring.profiles.active
SERVER_PORT                 ↔  server.port
```

## 13. DataSource·트랜잭션 전략

```text
[업무 WAR — local]
  dataSource (Primary)          → h2:mem:nsight_{code}     업무 SQL
  transactionLogDataSource      → h2:file .../nsight_om    TCF_TX_LOG
  transactionControlDataSource  → h2:file .../nsight_om    TCF_TRANSACTION_CONTROL
  timeoutPolicyDataSource       → h2:file .../nsight_om    TCF_SERVICE_TIMEOUT_POLICY

[tcf-om — local]
  dataSource (Primary)          → h2:file .../nsight_om    OM+세션+거래로그 통합
  transaction-log separate      → false

[tcf-batch]
  dataSource                    → h2:file .../nsight_om    OM_*_STATUS only
  transaction-log-enabled       → false

[tcf-gateway]
  dataSource                    → h2:file .../gateway-route
  sessionDataSource             → SESSIONDB (nsight_om TCP/file)
```

| 모듈 | Hikari auto-commit | Spring TX default |
|------|-------------------|-------------------|
| 업무 WAR | `false` | 5s |
| tcf-om | `true` | 5s |
| tcf-batch | default | — |

거래로그 INSERT는 업무 TX와 **별도 autocommit** — [온라인처리.md](온라인처리.md)

| `OmDatabaseMigration` | tcf-om | 0 | MERGE 시드, 카탈로그 |

| `BatchDatabaseMigration` | tcf-batch | — | 배치 테이블 |
| `DashboardCollectStartupRunner` | tcf-batch | 50 | 기동 1회 4 Job 수집 |

**스키마 DDL:**

| 대상 | 방식 |
|------|------|
| OM 테이블 | `spring.sql.init` + `schema.sql` |
| `TCF_TX_LOG` | `TransactionLogSchemaInitializer` @PostConstruct |
| 거래통제·Timeout | `TcfTransactionControlConfiguration` 등 |
