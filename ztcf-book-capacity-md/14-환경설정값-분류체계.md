## 절별 상세

### 14.1 인프라 설정

본 절(**인프라 설정**)은 분류체계 영역에서 **설정 분류** 관점의 NSIGHT 1차 표준을 정의합니다. 피크 **720 TPS**·p95 **3초 이하**·8 vCPU / 32GB 기준으로 권고값을 제시하며, 최종값은 선도개발·성능시험(360/720/1080) 실측으로 확정합니다.

#### NSIGHT 권고값

| 항목 | 설정 분류 권고 |
|------|----------------|
| 기준 VM | 8 vCPU / 32GB |
| TPS | 360/720/1080 |
| 설정 파일 | 환경셋팅(최종) |
| 핵심 | 7개 영역 |

#### 설정 예시

**설정 파일**: `환경셋팅(최종)` · **핵심 항목**: 7개 영역

#### 검증 기준

| 시나리오 | 합격 기준 |
|----------|----------|
| 360 TPS | p95≤3s, CPU≤70%, 오류율≤1% |
| 720 TPS | Thread/Pool 고갈 없음, Hikari Pending=0 |
| 1,080 TPS | Fail-fast·Timeout 정상, CB 동작 확인 |
| AP 1대 Down | L4 제외·잔여 TPS≥목표 |
| 센터 장애 | 잔여 센터 TPS≥720 |

#### 주의사항

- **피크 기준** 설계 — 평균 TPS만으로 설정 확정 금지
- **세션 ≠ TPS** — 세션 증가는 Heap·복제 부담, TPS와 별도 산정
- **Thread ≠ Pool** — Pool ≤ Thread×30~40%, DB Session 총량 검증
- **안쪽 Timeout 짧게** — SQL(3s) < Tx(5s) < Proxy(10s) < Web(15s)

### 14.9 공통값과 업무별 값

본 절(**공통값과 업무별 값**)은 분류체계 영역에서 **설정 분류** 관점의 NSIGHT 1차 표준을 정의합니다. 피크 **720 TPS**·p95 **3초 이하**·8 vCPU / 32GB 기준으로 권고값을 제시하며, 최종값은 선도개발·성능시험(360/720/1080) 실측으로 확정합니다.

#### NSIGHT 권고값

| 항목 | 설정 분류 권고 |
|------|----------------|
| 기준 VM | 8 vCPU / 32GB |
| TPS | 360/720/1080 |
| 설정 파일 | 환경셋팅(최종) |
| 핵심 | 7개 영역 |

#### 설정 예시

**설정 파일**: `환경셋팅(최종)` · **핵심 항목**: 7개 영역

#### 검증 기준

| 시나리오 | 합격 기준 |
|----------|----------|
| 360 TPS | p95≤3s, CPU≤70%, 오류율≤1% |
| 720 TPS | Thread/Pool 고갈 없음, Hikari Pending=0 |
| 1,080 TPS | Fail-fast·Timeout 정상, CB 동작 확인 |
| AP 1대 Down | L4 제외·잔여 TPS≥목표 |
| 센터 장애 | 잔여 센터 TPS≥720 |

#### 주의사항

- **피크 기준** 설계 — 평균 TPS만으로 설정 확정 금지
- **세션 ≠ TPS** — 세션 증가는 Heap·복제 부담, TPS와 별도 산정
- **Thread ≠ Pool** — Pool ≤ Thread×30~40%, DB Session 총량 검증
- **안쪽 Timeout 짧게** — SQL(3s) < Tx(5s) < Proxy(10s) < Web(15s)

## 환경설정값 분류체계 — 실무 요약

본 장은 **분류체계**(설정 분류)의 핵심을 23장(JVM)과 동일한 깊이로 요약합니다.

### E2E 용량산정 체인

```
[1] 사용자 21,600 → [2] 동시요청자 1,080~3,240
→ [3] TPS 360/720/1080 → [4] AP 8대 권장
→ [5] Thread 400~500 → [6] Pool 50/60
→ [7] DB Session Σ(AP×Pool) → [8] 장애 잔여 TPS≥720
```

### NSIGHT 1차 표준 요약

| 항목 | NSIGHT 1차 권고 |
|------|----------------|
| 사용자 | 21,600명 (3600×6) |
| 세션 | 26,000~28,000 |
| TPS | 360 / 720 / 1080 |
| VM | 8 vCPU / 32GB |
| VM당 TPS | 250 |
| p95 | 3초 이하 |

### Timeout 계층

| 계층 | 권고 | 설정 위치 |
|------|------|----------|
| SQL | 3초 | MyBatis / Mapper |
| Pool 획득 | 3초 | Hikari connectionTimeout |
| Transaction | 5초 | @Transactional |
| TCF ServiceId | 4~5초 | TCF 설정 |
| Proxy | 10초 | Apache / Gateway |
| WebTop | 15초 | 단말 |
| L4 Idle | 120초 | L4 Profile |

### 산정 공식

```
동시요청자 = 전체사용자 × 동시요청률
TPS = 동시요청자 ÷ 목표응답시간(3초)
AP 대수 = ⌈TPS ÷ 250⌉ (A-A 배치)
Thread = AP당TPS × 1.2초 × 1.2
Pool = max(30, min(TPS×0.15×1.3, Thread×30%))
```
