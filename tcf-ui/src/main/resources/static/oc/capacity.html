DB Pool (CAP-050)

DB Pool 산정 공식 보기

VM당 Hikari maximumPoolSize — 4단계 산정

  1. AP TPS = 목표 TPS ÷ 배포 AP 대수 A-A 시 센터당 필요 AP×2 · CAP-030
  2. ② 산출 Pool = ⌈AP TPS × DB 점유(초) × DB 비율 × Pool 안전계수⌉
  3. ③ 상한 Pool = ⌈VM당 WAS Thread × Thread→DB 비율⌉ CAP-040
  4. ⑤ 용량 권장 = min(②, ③) TPS·Thread가 허용하는 용량값 (운영 최소 미포함)
  5. 운영 최소 Pool/VM = 입력값 30 Hikari 운영 하한
  6. ④ 배포 Pool/VM = max(운영 최소, ⑤) maximumPoolSize · 프로파일 cap 가능

현재 입력: 산출 = AP TPS × — · 상한 = Thread × — · 최소 —

최소 Pool/VM 계산식

⑤ = min(②, ③) · ④ = max(30, ⑤)

현재 최소 Pool/VM = 30 (위 입력란과 동기화)
