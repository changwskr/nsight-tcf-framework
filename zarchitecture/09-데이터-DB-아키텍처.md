# 09. 데이터·DB 아키텍처

> **범위:** RDW, ADW, SESSIONDB, OMDB, LOGDB, Gateway DB, JWT DB, Pool·MyBatis  
> **관련:** [zman/19-DB-테이블.md](../zman/19-DB-테이블.md) · [zdoc/테이블정보.md](../zdoc/테이블정보.md)

---

## 1. 개요

### 1.1 DB 분리 원칙

```
업무 SQL → RDW (실시간) / ADW (분석)
운영 SQL → OMDB
세션 SQL → SESSIONDB
로그 SQL → LOGDB
Gateway → gateway-route DB
JWT → JWT schema
```

**업무 DB와 운영 DB 역할 분리** — DAO는 해당 Pool만 사용.

---

## 2. 논리 DB 맵

| DB (논리) | 용도 | 로컬(H2) 경로 |
|-----------|------|---------------|
| **RDW** | 실시간 조회, Single View | 업무 WAR별 |
| **ADW** | 분석·집계 (온라인 영향 분리) | (목표) |
| **SESSIONDB** | Spring Session, TCF_USER_SESSION | tcf-om H2 공유 |
| **OMDB** | 사용자, Catalog, 통제, 코드 | data/nsight-txlog/nsight_om |
| **LOGDB** | TCF_TX_LOG, OM_AUDIT_LOG | data/nsight-txlog |
| **Gateway DB** | TCF_GATEWAY_ROUTE, TCF_GATEWAY_TX_LOG | data/gateway-route |
| **JWT DB** | Token, Refresh, Denylist | tcf-jwt schema |
| **FILEDB** | UD_FILE, UD_DOWNLOAD_LOG | OMDB 내 |

## 3. TCF 통제 테이블

| 테이블 | 역할 | 조회 주체 |
|--------|------|-----------|
| OM_SERVICE_CATALOG | serviceId 마스터 | STF, OM Admin |
| TCF_TRANSACTION_CONTROL | Header 7 Allow-List | STF |
| TCF_SERVICE_TIMEOUT_POLICY | serviceId별 timeout | STF, TimeoutExecutor |
| TCF_IDEMPOTENCY_KEY | 중복 방지 | STF |
| TCF_SERVICE_AUDIT_POLICY | 감사 대상 | STF, ETF |

## 4. 로그 테이블

| 테이블 | 역할 |
|--------|------|
| TCF_TX_LOG | 거래 마스터 (PROCESSING→SUCCESS/FAIL/TIMEOUT) |
| TCF_TX_STEP_LOG | 단계별 (SQL ID 연계) |
| TCF_ERROR_LOG | 시스템 오류 |
| OM_AUDIT_LOG | 감사 |
| OM_ACCESS_LOG | 접근 |

## 9. MyBatis·DAO 아키텍처

### 9.1 계층

```
Service → DAO → @Mapper interface → XML/Annotation SQL
```

### 9.2 원칙

- DAO/Mapper = **SQL만**
- SQL ID = TCF_TX_STEP_LOG 연계
- [zdoc/mybatisNaming.md](../zdoc/mybatisNaming.md) 준수

### 9.3 Connection Pool

HikariCP — Pool **분리**:

| Pool | DB |
|------|-----|
| RDW Pool | 업무 조회 |
| ADW Pool | 분석 (목표) |
| SESSION Pool | Spring Session |

설정: `application-datasource.yml`, profile별 `tcf-cicd`
