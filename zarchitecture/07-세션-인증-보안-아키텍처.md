# 07. 세션·인증·보안 아키텍처

> **범위:** SESSIONDB, Spring Session JDBC, tcf-jwt, OM Auth, Gateway 세션 관문  
> **관련:** [zman/10-세션관리.md](../zman/10-세션관리.md) · [zman/11-JWT-SSO인증.md](../zman/11-JWT-SSO인증.md)

## 1. 개요

NSIGHT는 **두 가지 인증 경로**를 지원한다.

| 경로 | 메커니즘 | 용도 |
|------|---------|------|
| **세션 (주)** | JSESSIONID + SESSIONDB | WebTop, tcf-uj, OM Admin |
| **JWT (보조)** | Bearer RS256 | API·SSO·Resource Server |

Gateway는 **JWT를 사용하지 않음** — SESSIONDB 쿠키 4단계 검증만.

---

## 2. 세션 아키텍처

### 2.1 원칙

> 세션 원천 = **SESSIONDB** (Spring Session JDBC). WAS 메모리 세션 ❌

```
브라우저 (JSESSIONID / NSIGHTSID Cookie)
  → Apache [Sticky]
  → [tcf-gateway] Cookie Relay
  → 업무 WAR / tcf-om
  → Spring Session JDBC Filter
  → STF: header.user ↔ session user
  → SESSIONDB
```

### 2.2 다중 WAR 세션 공유

모든 WAR·Gateway가 **동일 SESSIONDB** → 로그인 1회, 전 업무 Cookie 공유

DeltaManager(WAS 메모리) ❌ → JDBC Session ✅

---

## 3. 세션 생성

### 3.1 로그인 흐름

```
POST /om/online  serviceId=OM.Auth.login
  → OmAuthHandler → OmAuthFacade
  → OM_USER 검증 (비밀번호)
  → HttpSession 생성 → Spring Session JDBC 저장
  → TCF_USER_SESSION INSERT (ACTIVE)
  → Set-Cookie: JSESSIONID
```

SSO: `OM.Auth.ssoLogin` → tcf-jwt `JWT.Auth.ssoIssue` (선택)

### 3.2 테이블

| 테이블 | 용도 |
|--------|------|
| SPRING_SESSION | Spring Session PK, expiry |
| SPRING_SESSION_ATTRIBUTES | 세션 속성 Blob |
| TCF_USER_SESSION | userId, branch, channel, STATUS, loginTime |

---

## 4. 세션 검증 (STF)

업무 요청 시:

```
POST /sv/online + Cookie + header(user, branch, channelId)
  ① Session Filter: SPRING_SESSION 조회, 만료
  ② STF: header user ↔ session user
  ③ branch, channelId 일치
  ④ 기능권한 (OM_FUNCTION_AUTH)
  ⑤ 중복 로그인 정책 (OM.Session.*)
  ⑥ Idle / Absolute Timeout
  ⑦ Dispatcher
```

---

## 5. Timeout 유형

| 유형 | 설명 | 설정 |
|------|------|------|
| **Idle** | 미사용 시간 | spring.session.timeout |
| **Absolute** | 로그인 후 최대 | OM 정책 |

세션 오류: E-TCF-SES-* (없음, 만료, 불일치, 중복)

## 7. JWT 아키텍처 (tcf-jwt)

### 7.1 모듈

| 항목 | 값 |
|------|-----|
| 모듈 | tcf-jwt |
| 포트 | 8110 |
| BC | JWT |

### 7.2 serviceId

| serviceId | 설명 |
|-----------|------|
| JWT.Auth.login | Access + Refresh 발급 |
| JWT.Auth.ssoIssue | OM SSO 후 JWT (내부) |
| JWT.Auth.refresh | Refresh Rotation |
| JWT.Auth.revoke | Access Denylist |
| JWT.Auth.logout | 전체 폐기 |

### 7.3 테이블

| 테이블 | 역할 |
|--------|------|
| TCF_JWT_TOKEN | Access 이력 |
| TCF_REFRESH_TOKEN | Refresh (해시) |
| TCF_TOKEN_DENYLIST | 폐기 JTI |
| OM_USER | 사용자 원장 |

### 7.4 JWK

`GET /.well-known/jwks.json` — RS256 공개키 (Resource Server)

### 7.5 OM 연동

`OmJwtSsoClient` → `JWT.Auth.ssoIssue`

Gateway: `JwtProxyController` → POST /jwt/online

## 8. 인증 vs 권한 vs 거래통제

| | 질문 | 처리 |
|---|------|------|
| **인증** | 누구인가 | 세션/JWT, OM.Auth.login |
| **권한** | 기능 쓸 수 있는가 | OM_FUNCTION_AUTH, STF |
| **거래통제** | Header 7 조합 허용? | TCF_TRANSACTION_CONTROL, STF |

## 9. 보안 설정

### 9.1 application-session.yml

```yaml
spring.session.store-type: jdbc
spring.session.timeout: 30m
```

### 9.2 Gateway

```yaml
nsight.gateway.auth.login-required: true
nsight.gateway.auth.session-validation.enabled: true
server.servlet.session.tracking-modes: []
```

### 9.3 비밀

- prod: `${NSIGHT_*}` placeholder, Git 미포함
- JWT RS256 키: JwtKeyConfiguration
