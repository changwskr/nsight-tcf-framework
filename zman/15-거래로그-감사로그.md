## 전체 흐름 (15.4)

```
STF → TCF_TX_LOG INSERT (PROCESSING)
Handler/DAO → (선택) Step Log, SQL 시간
ETF → TCF_TX_LOG UPDATE (SUCCESS/FAIL/TIMEOUT/UNKNOWN)
    → OM_AUDIT_LOG (감사 대상)
    → Metrics
```

**Handler에서 로그 직접 기록 ❌** — STF/ETF 자동.

## 거래 상태 (15.6)

| 상태 | 의미 |
|------|------|
| PROCESSING | STF 시작 |
| SUCCESS | 정상 |
| FAIL | 오류 |
| TIMEOUT | 시간 초과 |
| **UNKNOWN** | 결과 불명 (등록·연계) — **실패 단정 금지** |

## TCF_TX_LOG (15.7)

GUID, TraceId, TransactionId, serviceId, transactionCode, user, channel, branch, start/end, elapsed, errorCode, status

## TCF_TX_STEP_LOG (15.7)

- 구간별 Step  
- **MyBatis Mapper ID** 연계 필수  

## OM_AUDIT_LOG (15.8)

userId, IP, serviceId, action, before/after, auditLevel

## 로그 유형 (15.5)

- Apache Access Log (WEB) ↔ TCF TX Log **GUID 연계**
- MDC: guid, traceId, serviceId (`logback-spring.xml`)

## OM 조회 (15.x)

- `OmTransactionLogHandler` — `OM.TransactionLog.*`
- `OmAuditLogHandler` — `OM.AuditLog.*`

## 코드베이스
