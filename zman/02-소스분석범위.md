## README 4원칙 (2.4)

1. TransactionHandler + **serviceId**
2. **STF/TCF/ETF** 파이프라인
3. **업무 독립 WAR**
4. **bootRun + ztomcat**

## 업무 WAR 6계층 (2.7)

```
com.nh.nsight.marketing.{업무}
├── application/ (service, rule, scheduler)
├── client/
├── config/
├── entry/ (handler, facade, web)
├── persistence/ (dao, mapper)
└── support/
```

## TCF 처리 흐름 — 분석 핵심 (2.8)

```
POST /{businessCode}/online
→ tcf-web → TCF.process()
   STF → OnlineTransactionTimeoutExecutor → TransactionDispatcher → Handler
   → Facade → Service → Rule → DAO/Mapper → ETF
```

## 모듈별 분석 질문 (2.5~2.13)

| 모듈 | 분석 질문 |
|------|-----------|
| tcf-core | STF/Dispatcher/ETF, TransactionHandler.serviceIds() |
| tcf-web | /online, 거래로그 DB |
| tcf-om | Catalog, 거래통제, 24 도메인 Handler |
| tcf-gateway | businessCode Relay, SESSIONDB |
| tcf-jwt | 발급·JWKS, Internal Call (검증=gateway) |
| tcf-eai | TcfServiceClient, Header 전파 |
| tcf-batch | BAT-BATCH-001~004 |
| tcf-ui/uj | OM Relay, Gateway 테스트 |
