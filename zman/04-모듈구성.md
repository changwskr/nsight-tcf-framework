## 모듈 트리 (4.3)

```
nsight-tcf-framework
├── docs, zdoc, zman
├── tcf-util → tcf-core → tcf-web → tcf-cache(선택)
├── tcf-om, tcf-batch, tcf-ui, tcf-uj
├── tcf-gateway, tcf-jwt, tcf-eai
├── tcf-cicd, tcf-scripts, ztomcat
└── ic, pc, ms, sv, pd, eb, ep, ss, mg (-service)
```

## 의존 방향 (4.5)

```
tcf-util → tcf-core → tcf-web → tcf-cache(선택)
                              → 업무 WAR / tcf-om / tcf-batch / tcf-jwt / tcf-gateway
```

순환·역방향 의존 금지.

## 공통 Framework (4.6)

| 모듈 | 역할 |
|------|------|
| tcf-util | 순수 유틸 |
| tcf-core | STF, Dispatcher, ETF, TransactionHandler |
| tcf-web | /online, TcfGateway, 거래로그 |
| tcf-cache | EhCache, TcfCacheSupport |

## 업무 WAR (4.7~4.8)

- **9개**: ic, pc, ms, sv, pd, eb, ep, ss, mg
- `om-service`: 레거시 참고 → **tcf-om** 사용
- 6계층 + `entry/handler → facade → service → rule → dao`

## tcf-eai (4.10)

| 항목 | 내용 |

| API | `TcfServiceClient`, `StandardRequestBuilder` |
| 문서 | `zdoc/서비스간연동.md`, `tcf-eai/README.md` |

## 모듈 원칙 (4.16)

1. 공통·업무 분리  
2. WAR 간 tcf-eai  
3. OM = 기준정보 원장  
4. Gateway = Relay + 1차 검증
