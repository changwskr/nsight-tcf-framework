## 핵심 결론

> **Allow-List**: 등록된 Header 7조합만 실행. 권한 있어도 **미등록 → 차단**.

---

## Header 7항목 (13.4)

serviceId, transactionCode, businessCode, serviceName, user, channelId, branch

**제외**(별도): GUID, TraceId, Timeout, Role

## 권한 vs 거래통제 (13.5)

| | 질문 |
|---|------|
| **권한** | 사용자가 기능 **쓸 수 있는가** |
| **거래통제** | Header 조합 거래가 **허용 등록**됐는가 |

## Catalog vs TC (13.6)

| | Catalog | TCF_TRANSACTION_CONTROL |
|---|---------|-------------------------|
| 질문 | serviceId **존재**? | 이 **조합** 허용? |

## 처리 위치 (13.3)

```
STF.preProcess()
  → Header 7 추출
  → TCF_TRANSACTION_CONTROL 조회 (Cache 가능)
  → 허용 → Dispatcher / 거부 → E-TCF-CTL-*
```

Gateway: **1차** 가능, STF: **최종**

## 테이블 (13.7)

**TCF_TRANSACTION_CONTROL** — Allow-List 컬럼 = Header 7 + ACTIVE

## 판단 순서 (13.8)

1. 7항목 추출 · 필수 확인  
2. Catalog 등록  
3. TC 매칭 (와일드카드 규칙 13.9)  
4. ACTIVE  
5. Dispatcher 또는 차단  

## 실패 예 (13.11)

| 케이스 | 결과 |
|--------|------|
| 미등록 serviceId | CTL-0001 |
| branch 999999 미등록 | CTL-0001 |
| businessCode 불일치 | HDR-* |
