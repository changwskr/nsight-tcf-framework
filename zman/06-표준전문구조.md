## Endpoint

```
POST /{businessCode}/online
Content-Type: application/json; charset=UTF-8
Cookie: JSESSIONID=...
```

## Header 7 — 거래통제·STF (6.5)

serviceId, transactionCode, businessCode, serviceName, user, channelId, branch

추적: guid, traceId, transactionId, requestTime, screenId, menuId, idempotencyKey

## Body (6.6~6.7)

업무별 자유. 페이징: `body.page` + `body.list` — **TCF Core 비관여**, Rule/DAO (6.12)

## Result (6.9)

resultCode, errorCode, message, detailMessage, errorType, retryable, causeServiceId, causeSqlId

## 실패 예 (6.11)

| 유형 | errorCode | errorType |
|------|-----------|-----------|
| Header | E-TCF-HDR-0001 | HEADER_VALIDATION |
| 거래통제 | E-TCF-CTL-0001 | TRANSACTION_CONTROL |
| Timeout | E-TCF-TIME-0001 | TIMEOUT |

## HTTP vs JSON Header (6.13)

| HTTP | JSON header | body |
|------|-------------|------|
| Content-Type, Cookie, Authorization | serviceId, user… | 업무 데이터 |

## STF 검증 (6.14)

필수 Header, Path↔businessCode, Prefix↔businessCode, serviceName default (`StandardRequestBuilder`)

## Java DTO (6.17) — tcf-core

`StandardRequest<T>`, `StandardResponse<T>`, `StandardHeader`, `Result`  
Body: 주로 `Map<String,Object>`

## 거래로그 (6.15)

Header → TCF_TX_LOG (GUID, TraceId, serviceId…)
