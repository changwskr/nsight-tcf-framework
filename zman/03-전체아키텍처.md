## 핵심 결론

NSIGHT = **GSLB → L4 → Apache → [Gateway] → 업무 WAR → TCF → Handler → MyBatis → RDW/ADW/SESSIONDB/LOGDB**

목표: 사용자 **36,000** / 세션 **43,200** / TPS **720** / P95 **3초** / 가용성 **99.99%**

## 논리 아키텍처 (3.3)

```
[WebTopSuite / API / tcf-ui]
  → GSLB → L4 → Apache (SSL, Proxy, LB, Sticky)
  → [tcf-gateway]  ← Application Gateway (선택)
  → 업무 WAR / tcf-om  (/sv/online, /ic/online, /om/online)
  → tcf-web (Controller) → tcf-core (STF/Dispatcher/ETF)
  → Handler → Facade → Service → Rule → DAO
  → RDW | ADW | SESSIONDB | LOGDB | OMDB
```

## 계층별 역할 (3.4)

| 계층 | 역할 |
|------|------|
| Apache | SSL, Reverse Proxy, URL Routing, LB, Logging |
| Tomcat/Spring Boot | Web Container, Session Cookie, JDBC Session |
| tcf-gateway | businessCode 라우팅, Cookie/JWT 1차 검증 |
| tcf-web | HTTP `/online` 진입 |
| tcf-core | STF/TCF/ETF |
| MyBatis/HikariCP | SQL, Connection Pool 통제 |

## TCF 실행 (3.5)

단일 진입 `TCF.process()` — URL마다 Controller를 늘리지 않음.

## 애플리케이션 6계층 (3.6)

Handler(진입) → Facade(트랜잭션) → Service(흐름) → Rule(규칙) → DAO(SQL)

## 데이터 (3.7)

| DB | 용도 |
|----|------|
| RDW | 실시간 조회, Single View |
| ADW | 분석·집계 (온라인 영향 분리) |
| SESSIONDB | Spring Session JDBC |
| LOGDB | TCF_TX_LOG, OM_AUDIT_LOG |
| OMDB | 기준정보 원장 |

## 세션·Gateway·OM·JWT (3.8~3.11)

- 세션 원천: **SESSIONDB** (WAS 메모리 아님)
- Gateway: 인프라 대체 ❌, **업무 Relay** ✅
- OM: 운영 기준정보 **원장**
- JWT: 인증 수단, TCF는 통제·로그 계속 담당

## 배포 (3.12)

| 모드 | 용도 |
|------|------|
| bootRun | 개발, 모듈별 JVM |
| ztomcat | Tomcat 8080 다중 Context 통합 검증 |
| 운영 Tomcat | 17 WAR (목표) |

WAR: `WEB-INF/lib`에 tcf-* JAR 내장.

## 성능·용량 (3.14)

- Connection Pool = **자원 통제** (HikariCP)
- Query Timeout, Online Timeout 다층 (14장)
- Cache = 기준정보만 (16장)

## 아키텍처 원칙 (3.15)

1. 표준 전문 + serviceId Dispatcher  
2. 공통 STF/ETF, 업무 Handler만 분리  
3. 업무 WAR 간 **HTTP/JSON** (tcf-eai)  
4. OM에서 Catalog·통제·Timeout 운영  

## 코드베이스 매핑

| 계층 | 모듈/클래스 |
|------|-------------|
| HTTP | `tcf-web` — OnlineTransactionController |
| TCF | `tcf-core` — TCF, STF, TransactionDispatcher, ETF |
| 업무 | `*-service/entry/handler/*Handler` |
| Gateway | `tcf-gateway` |
| OM | `tcf-om` |
