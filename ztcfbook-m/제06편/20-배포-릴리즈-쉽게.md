# 제20장. 배포·릴리즈 쉽게

| 항목 | 내용 |
| --- | --- |
| **편** | 제6편 |
| **상태** | 집필 완료 |
| **원본** | [ztcfbook 제20장](../ztcfbook/제06편/20-CICD-릴리즈-DR.md) |

---

## 20.1 배포 단위 = WAR

TCF는 **WAR 파일** 하나씩 Tomcat에 올립니다.

| | |
| --- | --- |
| 배포함 | `sv.war`, `om.war`, `gw.war` … |
| 따로 배포 안 함 | tcf-core, tcf-web (WAR **안에** 포함) |

```bash
gradle :sv-service:bootWar
```

비유: tcf-core/web = **부품**, WAR = **완성 차**.

---

## 20.2 배포 순서 (한 줄)

```text
코드 merge → build·test → bootWar → 저장소 업로드
→ Tomcat에 WAR 교체 → Health → Smoke curl
→ OM 배포 이력 기록
```

배포 **전** 꼭 볼 것:

| # | 확인 |
| --- | --- |
| 1 | `gradlew build` **통과** |
| 2 | OM Catalog·Route **seed diff** |
| 3 | **이전 WAR(N-1)** 백업 (롤백용) |
| 4 | Secret은 yml 평문 **X** |

---

## 20.3 CI/CD Pipeline

28장 **한 줄**을 조금만 펼치면:

| Stage | 뭐 하나 |
| --- | --- |
| build | compile + **bootWar** |
| test | 단위·통합 (21장) |
| quality | SonarQube 등 |
| deploy | dev → stg → **prd(승인)** |
| verify | Health + **Smoke 3~5건** |

원칙: **테스트 통과 전에는 배포 안 함.**  
`-x test` 배포는 **긴급 hotfix** 때만.

---

## 20.4 Git 브랜치 (외울 것 4개)

```text
feature/*  →  develop  →  release/*  →  main + Tag v1.2.3
```

| 브랜치 | 배포 |
| --- | --- |
| feature | **로컬만** |
| develop | dev (자동) |
| release | stg |
| main | **운영** (Tag 필수) |

---

## 20.5 롤백 = WAR 되돌리기

문제 생기면:

1. **직전 WAR(N-1)** 다시 올림
2. Tomcat restart
3. Health + Smoke
4. OM `Deploy` 이력 남김

목표: **30분 안**에 WAR 롤백 (설계 기준).

⚠ DB migration이 **되돌릴 수 없으면** WAR만 롤백해도 DB가 안 맞을 수 있음 → **stg에서 먼저** 검증.

---

## 20.6 장애 나면 — guid부터

```text
1. 응답 resultCode / HTTP status
2. guid로 거래로그·Gateway 로그 검색
3. 최근 **배포·Catalog·Cache Evict** 이력
4. OM Dashboard (AP/DB/세션)
```

| 증상 | 먼저 볼 곳 |
| --- | --- |
| 401 Gateway | Route, 세션 DB, JWT |
| Catalog 없음 | tcf-om, data.sql |
| Dashboard 비어 있음 | **tcf-batch** 8098 |

---

## 20.7 ⚠️ 초보자 실수

| 실수 | |
| --- | --- |
| bootRun JAR를 운영에 배포 | **bootWar** |
| tcf-core만 고치고 sv WAR 안 rebuild | **영향 WAR 전부** rebuild |
| Smoke 없이 prod | **대표 serviceId curl** |
| prod yml에 비밀번호 평문 | **환경 변수·Vault** |

---

## 요약

- **WAR**가 배포 단위, tcf-core/web은 **안에 포함**
- **build → test → bootWar → deploy → Smoke**
- 롤백 = **N-1 WAR** + Health
- 장애 = **guid** 추적
