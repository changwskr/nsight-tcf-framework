## 30.1 4종 요약

| WAR | 코드 | 포트 | 특징 |
| --- | --- | --- | --- |
| eb-service | EB | 8089 | 이벤트 **발행** (Outbox) |
| ep-service | EP | 8090 | 이벤트 **수신** |
| ss-service | SS | 8093 | 시스템 샘플 |
| mg-service | MG | 8096 | 관리·설정 샘플 |

Handler·6계층 패턴은 **29장과 동일**합니다.

---

## 30.2 EB → EP 흐름 (이벤트 연습)

```text
EB.User.create  →  EB_EVENT(READY)
@Scheduled      →  POST ep/online (EP.UserEvent.receive)
성공            →  EB_EVENT(SENT)
```

**WAR끼리 Java import 금지** — HTTP로만.

```bash
gradle :eb-service:bootRun
gradle :ep-service:bootRun
# EB create 후 Scheduler가 EP 호출하는지 확인
```

---

## 30.3 ss · mg — 확장 템플릿

신규 거래 prefix:

- **SS.** — 시스템 공통 샘플
- **MG.** — 관리 도메인 샘플

ic/sv와 **똑같이** Handler 추가하면 됩니다.

---

## 30.4 ⚠️ 초보자 실수

| 실수 | |
| --- | --- |
| EB에서 EP Bean 직접 호출 | **HTTP / tcf-eai** |
| EP 안 띄우고 EB만 테스트 | **End-to-End** 필요 |
| 이벤트 = tcf-batch | **업무 Scheduler**는 각 WAR |

---

## 요약

- **EB/EP** = 이벤트 발행·수신 **연습용**
- **SS/MG** = 추가 WAR **템플릿**
- 9개 업무 WAR **전체 지도** 완성
