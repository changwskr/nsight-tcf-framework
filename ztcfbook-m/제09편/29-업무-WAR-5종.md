## 29.1 공통 패턴 — 9개 WAR 모두 같음

```text
POST /{업무코드}/online → Handler → Facade → … → Mapper
```

| 규칙 | |
| --- | --- |
| Controller | **없음** |
| serviceId | `{BC}.대상.행동` |
| WAR 간 호출 | **tcf-eai** |

---

## 29.2 포트·샘플 한 표

| WAR | 코드 | 포트 | 샘플 serviceId |
| --- | --- | --- | --- |
| ic-service | IC | 8082 | `IC.Sample.inquiry` |
| pc-service | PC | 8083 | `PC.Sample.inquiry` |
| ms-service | MS | 8085 | `MS.Sample.inquiry` |
| **sv-service** | **SV** | **8086** | **`SV.Customer.selectSummary`** |
| pd-service | PD | 8087 | `PD.Sample.inquiry` |

```bash
gradle :sv-service:bootRun
curl -X POST http://127.0.0.1:8086/sv/online \
  -H "Content-Type: application/json" \
  -d @tcf-ui/src/main/resources/sample-requests/sv-sample-inquiry.json
```

UI: http://localhost:8099/{code}/index.html

---

## 29.3 sv-service = 실습 기준 WAR

22~23장 전부 **sv-service** 기준입니다.

| 거래 | serviceId |
| --- | --- |
| 고객요약 | `SV.Customer.selectSummary` |
| 고객목록 | `SV.Customer.selectList` |
| IC 연동 | `SV.Integration.icSample` |

---

## 29.4 신규 거래 5단계 (복습)

1. serviceId·거래코드 설계
2. **OM Catalog** 등록
3. Handler `serviceIds()` + switch
4. Facade ~ Mapper
5. curl / tcf-ui 테스트

---

## 29.5 ⚠️ 초보자 실수

| 실수 | |
| --- | --- |
| SV 말고 다른 WAR로 첫 실습 | **SV가 샘플 최다** |
| 업무코드·포트 혼동 | **부록 A** 표 참고 |
| ic를 Java import | **tcf-eai** |
