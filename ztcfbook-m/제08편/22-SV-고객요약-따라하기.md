# 제22장. SV 고객요약 따라하기

| 항목 | 내용 |
| --- | --- |
| **편** | 제8편 |
| **상태** | 집필 완료 |
| **원본** | [ztcfbook 제22장](../ztcfbook/제08편/22-조회-거래-SV-고객요약.md) |

---

## 22.1 이 실습에서 할 일

**“고객 번호 넣으면 이름·등급 요약 보여 주기”** — NSIGHT에서 **가장 많이 베끼는** 샘플입니다.

| 항목 | 값 |
| --- | --- |
| serviceId | `SV.Customer.selectSummary` |
| 거래코드 | `SV-INQ-0001` |
| 주소 | `POST http://localhost:8086/sv/online` |

---

## 22.2 순서 (6단계)

```text
1. [19장] sv-service bootRun (8086)
2. JSON 파일 준비 (아래 예시)
3. curl 또는 tcf-ui로 POST
4. result.resultStatus = SUCCESS 확인
5. body에 고객 정보 있는지 확인
6. (선택) OM 거래로그 화면에서 GUID 검색
```

---

## 22.3 JSON 예시 (curl용)

`request.json` 파일로 저장:

```json
{
  "header": {
    "businessCode": "SV",
    "serviceId": "SV.Customer.selectSummary",
    "transactionCode": "SV-INQ-0001",
    "processingType": "INQUIRY",
    "channelId": "WEBTOP",
    "userId": "dev01",
    "branchId": "001234"
  },
  "body": {
    "customerNo": "CUST0000001"
  }
}
```

> 샘플 파일: `tcf-ui/src/main/resources/sample-requests/` 아래 JSON도 사용 가능합니다.

---

## 22.4 curl 명령

```bash
curl -X POST http://127.0.0.1:8086/sv/online \
  -H "Content-Type: application/json; charset=UTF-8" \
  -d @request.json
```

Windows PowerShell (파일 경로는 본인 PC에 맞게):

```powershell
curl.exe -X POST http://127.0.0.1:8086/sv/online `
  -H "Content-Type: application/json; charset=UTF-8" `
  -d "@request.json"
```

**기대:** HTTP 200 + `"resultStatus":"SUCCESS"` + `body`에 고객 필드.

---

## 22.5 코드는 어디에?

이미 저장소에 **샘플 구현**이 있습니다. 직접 만들 때는 **6계층 순서**로 파일을 추가합니다.

```text
SvCustomerHandler      → Facade.selectSummary()
  → Service → Rule (customerNo 필수?)
  → Dao → SvCustomerMapper.xml (SELECT)
```

**Controller 클래스는 만들지 않습니다.**

---

## 22.6 테스트 4종

| # | 입력 | 기대 |
| --- | --- | --- |
| 1 | 정상 customerNo | SUCCESS |
| 2 | customerNo 빈 값 | 검증 오류 |
| 3 | 없는 고객 | 업무 오류 (E-SV-…) |
| 4 | serviceId 오타 | Dispatcher 오류 |

---

## 22.7 tcf-ui로 (선택)

```bash
gradlew :tcf-ui:bootRun
```

브라우저 `http://localhost:8099` — SV 메뉴에서 **Customer Summary** 선택 후 실행.

---

## 22.8 ⚠️ 안 될 때

| 증상 | 확인 |
| --- | --- |
| Connection refused | sv-service **8086** 기동? |
| Handler not found | serviceId **오타**? |
| Catalog 차단 | local에서 통제 off 여부, serviceId 등록 |
| 404 on /sv/online | Context **/sv** 맞는지 |

---

## 요약

- **8086/sv/online** + JSON header/body.
- **6계층** 패턴의 **표준 복사본** = 이 거래.
- 성공하면 **다른 조회 거래**도 같은 방식으로 확장.
