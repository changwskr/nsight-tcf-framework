## 5.1 “이름 하나”가 전부 연결된다

NSIGHT에서는 **이름을 한 번 정하면** 아래가 **같은 줄**로 맞춰집니다.

```text
업무코드 SV
  → 주소 /sv
  → WAR sv.war
  → 패키지 ...marketing.sv
  → serviceId SV.Customer.selectSummary
  → 거래코드 SV-INQ-0001
  → Handler SvCustomerHandler
  → Mapper SvCustomerMapper
```

**비유:** 주민등록번호처럼, **SV로 시작하는 이름**은 “Single View 업무”라는 뜻입니다. 중간에 `CM`이 섞이면 Gateway·로그·OM이 **엉킵니다**.

---

## 5.2 꼭 지킬 것 5가지 (12가지 중 핵심)

| # | 규칙 | 쉬운 말 |
| --- | --- | --- |
| 1 | 업무코드 먼저 | SV, OM, IC … 2글자 |
| 2 | URL은 **기능 이름 아님** | `/sv/online` 하나 |
| 3 | **serviceId** = 실행 | `SV.무엇.어떤행동` |
| 4 | **거래코드** = 로그용 번호 | `SV-INQ-0001` |
| 5 | OM에 **등록된 것만** 실행 | Catalog 빼먹으면 운영에서 막힘 |

나머지(DTO, SQL ID, 오류코드…)는 거래 하나 만들면서 채우면 됩니다.

---

## 5.3 SV 고객요약 — 연결 예시

| 항목 | 값 |
| --- | --- |
| 업무코드 | SV |
| 호출 주소 | `POST http://localhost:8086/sv/online` |
| serviceId | `SV.Customer.selectSummary` |
| 거래코드 | `SV-INQ-0001` |
| Handler | `SvCustomerHandler` |
| Facade | `SvCustomerFacade` |

이 표를 **설계할 때 먼저** 채우고 코드를 짭니다. (8장 체크리스트)

---

## 5.4 ⚠️ 초보자 실수

| 실수 | 왜 문제? |
| --- | --- |
| 패키지를 `com.example.demo` | 업무코드와 **안 맞음** |
| serviceId `selectCustomer` | **SV.** 로 시작 안 함 |
| 거래코드 `SV001` | **형식** 틀림 |
| Catalog 등록 안 함 | 로컬만 되고 **운영 실패** |

---

## 요약

- 개발 표준 = **이름들이 한 줄로 이어지는 것**.
- **serviceId**(실행)와 **거래코드**(로그)는 **둘 다** 필요합니다.
- 코드 전에 **표 한 장**으로 이름을 맞추세요.
