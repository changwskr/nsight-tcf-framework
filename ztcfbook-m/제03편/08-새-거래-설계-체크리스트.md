## 8.1 코드 전에 할 일

**순서를 바꾸면** 나중에 이름·OM·화면을 **전부 고칩니다.**

```text
1. 화면/요건: "고객 요약 보여 주기"
2. serviceId 정하기: SV.Customer.selectSummary
3. 거래코드 정하기: SV-INQ-0001
4. OM Catalog 등록 요청
5. Handler ~ Mapper 구현
6. curl / tcf-ui 로 테스트
```

---

## 8.2 설계표 (복사해서 쓰기)

| 항목 | 내 값 |
| --- | --- |
| 업무코드 | |
| Endpoint | POST /__/online |
| serviceId | |
| transactionCode | |
| processingType | INQUIRY / CREATE / … |
| Handler 클래스 | |
| Timeout (초) | |
| 감사로그 Y/N | |

---

## 8.3 Header에 꼭 들어가는 것

최소한 아래는 **클라이언트(JSON)** 가 보냅니다.

| 필드 | 예 |
| --- | --- |
| businessCode | SV |
| serviceId | SV.Customer.selectSummary |
| transactionCode | SV-INQ-0001 |
| processingType | INQUIRY |
| channelId | WEBTOP |

GUID·TraceId는 **없으면 STF가 만들어** 줍니다.

---

## 8.4 ⚠️ 초보자 실수

| 실수 | 결과 |
| --- | --- |
| 코드부터 작성 | Catalog·거래코드 **나중에 꼬임** |
| processingType INQUIRY인데 등록 SQL | 감사·트랜잭션 **과함** |
| 거래코드만 바꿔 재사용 | 로그·통계 **오염** |

---

## 요약

- **설계표 → OM 등록 → 코드** 순서.
- serviceId·거래코드·processingType **세트**로 정하기.
