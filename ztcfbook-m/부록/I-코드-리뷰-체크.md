## 리뷰 = “운영 표준 맞나?”

취향이 아니라 **6계층·serviceId·로그·OM 등록**을 봅니다.

리뷰어가 물어볼 **4가지**:

| # | 질문 |
| --- | --- |
| 1 | 로그만 보고 **어느 거래**인지 알 수 있나? |
| 2 | serviceId만 보고 **어느 Handler**인지 알 수 있나? |
| 3 | 오류코드만 보고 **뭘 고칠지** 알 수 있나? |
| 4 | SQL ID만 보고 **어느 Mapper**인지 알 수 있나? |

---

## MR 승인 전 15초 (15항)

| □ | 핵심 |
| --- | --- |
| □ | 업무코드·패키지·WAR **일치** |
| □ | serviceId `SV.대상.행동` |
| □ | 거래코드 `SV-INQ-0001` |
| □ | **6계층** (Handler→Facade→…→Mapper) |
| □ | 표준 JSON (header + body / result) |
| □ | Request·Response DTO **분리** |
| □ | 필수값·형식 **Validation** |
| □ | SQL = Mapper XML, `#{}` 바인딩 |
| □ | 변경 거래 **@Transactional** (Facade) |
| □ | **Timeout** 설정 |
| □ | `BusinessException` + 표준 오류코드 |
| □ | 로그에 **GUID·serviceId** |
| □ | 권한·**마스킹**·Injection 방지 |
| □ | **테스트** (Handler·통합) |
| □ | OM **Catalog·통제** 등록 준비 |

부록 H와 **겹치는 항목** — H 통과 후 I로 한 번 더.

---

## 바로 반려 (No)

| 패턴 | 이유 |
| --- | --- |
| Handler에서 Mapper **직접** | 계층 위반 |
| Java 안에 SQL 문자열 | 추적 불가 |
| `RuntimeException("오류")` | 오류코드 없음 |
| `${}` 사용자 입력 | SQL Injection |
| Catalog 없이 운영 배포 | **실행 차단** |

## 판정 4종

| 결과 | 의미 |
| --- | --- |
| **승인** | Merge OK |
| **조건부** | 주석·포맷만 — Merge 후 보완 |
| **보완** | 테스트·표준 위반 — **수정 후 재리뷰** |
| **반려** | 구조·보안 — **재설계** |
