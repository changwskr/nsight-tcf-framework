## 한 줄로

이름 하나가 **업무코드 → URL → 패키지 → ServiceId → Handler → SQL → 로그**까지 이어져야 합니다.

```text
SV → /sv/online → com.nh.nsight.marketing.sv
  → SV.Customer.selectSummary → SvCustomerHandler → Mapper XML
```

21개 주제는 **znsight-man/명명규칙-01~21** 에 상세히 있습니다. 여기서는 **초보가 자주 쓰는 것**만 압축합니다.

## 신규 거래 만들 때 순서

1. **07** ServiceId 정하기 → Handler `serviceIds()`
2. **08** 거래코드 → Header `transactionCode`
3. **09** Handler/Facade 클래스명
4. **12** Mapper XML + SQL id
5. **17** OM Catalog·화면 연결
6. **14** 업무 오류코드 `E-SV-BIZ-xxxx`

## ⚠️ 초보자 실수

| 실수 | |
| --- | --- |
| ServiceId와 거래코드 **혼동** | ServiceId=실행 키, 거래코드=감사 |
| `selectCustomer` vs `SV.Customer.selectSummary` | **ServiceId 형식** 틀림 |
| Mapper namespace를 DAO와 다르게 | MyBatis **바인딩 실패** |
| 21주제 전부 외우려 함 | **07·08·09·12** 먼저 |
