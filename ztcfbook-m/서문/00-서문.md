## 0.1 이 책은 누구를 위한 건가요?

**Java로 Spring Boot 프로젝트를 조금 해본 적 있는데, NSIGHT TCF는 처음**인 분을 위한 책입니다.

`@RestController`에 `@GetMapping("/users")`처럼 **URL마다 API를 만드는 방식**은 익숙할 수 있습니다. NSIGHT는 그 방식을 **업무 서버(WAR) 안에서는 쓰지 않습니다.** 대신:

- 주소는 거의 항상 **`POST /sv/online`** 처럼 **한 종류**이고
- JSON 안의 **`serviceId`** 로 “지금 어떤 기능을 실행할지” 정합니다.

이 차이만 이해해도 TCF 문서의 절반은 통과입니다. 나머지는 “요청이 들어오면 **누가** 검사하고, **누가** 업무 코드를 부르는지” 순서를 외우면 됩니다.

### 이 책에서 배우는 것

| 배울 내용 | 왜 필요한가 |
| --- | --- |
| TCF가 **REST와 다른 이유** | 팀 표준을 어기면 코드 리뷰에서 돌아옵니다 |
| **serviceId**, **거래코드** 이름 짓기 | OM(운영 화면)에 등록할 때 필요합니다 |
| **Handler → Facade → … → Mapper** 순서 | 파일을 어디에 만들지 정해 줍니다 |
| **로컬에서 SV 조회** 한 번 해보기 | “돌아가는 그림”을 몸으로 익힙니다 |

### 이 책에서 다루지 않는 것

서버실에 Apache를 설치하는 방법, DB 서버 운영, GitLab Runner 설치 같은 **인프라**는 다루지 않습니다. 개발 PC에서 `bootRun`으로 WAR를 띄우고 **표준 JSON으로 거래를 호출**하는 것까지만 다룹니다.

## 0.2 ztcfbook-m과 ztcfbook

저장소에는 TCF 책이 **두 층** 있습니다.

| 폴더 | 비유 | 언제 읽나 |
| --- | --- | --- |
| **`ztcfbook-m/`** (지금 책) | **입문 교재** | 첫 2~4주, 개념 + 실습 + **부록 A~N** |
| **`ztcfbook/`** (원본) | **개발자 백과** | 표준표, OM, CI/CD, Gap |

각 절 끝 **「📘 원본에서 더 보기」**는 ztcfbook **상세·운영 표**용입니다. 입문에 필요한 yml·명명·소스 안내는 **부록 G·M·N**에도 있습니다.

다른 폴더도 이름만 알아두면 됩니다.

```text
zguide/     → “sv-service 모듈은 이렇게 연다” (모듈 설명서)
znsight-man/ → 회사 개발 매뉴얼 전체 (78장)
docs/       → 클래스·설정까지 아주 자세한 기술 문서
```

**헷갈릴 때:** 동작·코드는 **실제 프로젝트 소스**, 절차는 **znsight-man**, 개념은 **ztcfbook-m → ztcfbook** 순으로 보면 됩니다.

## 0.3 꼭 기억할 단어 (미리보기)

처음부터 외울 필요는 없습니다. 1장에서 다시 나옵니다.

| 단어 | 한 줄 설명 |
| --- | --- |
| **TCF** | 온라인 거래를 **같은 방식**으로 처리하게 해 주는 프레임워크 |
| **serviceId** | 실행할 기능 이름 (예: `SV.Customer.selectSummary`) |
| **Handler** | serviceId를 받아 **Facade만** 호출하는 진입 클래스 |
| **STF / ETF** | 요청 **앞**·**뒤** 공통 처리 (검사, 로그, 응답 만들기) |
| **StandardRequest** | `{ "header": {...}, "body": {...} }` 형태의 JSON 요청 |
| **6계층** | Handler → Facade → Service → Rule → DAO → Mapper |

## 0.4 읽는 순서

```text
서문 (지금) → 1장 TCF가 뭐예요? → 2장 시스템 그림
→ 3장 요청이 지나가는 길 → 4장 6계층
→ (다음) ServiceId·Handler·실습
```

Spring Boot만 알고 TCF를 모른다면 **1장과 10장(원본)** 사이를 REST Controller로 채우지 마세요. 그게 가장 흔한 실수입니다.

---

## 요약

- NSIGHT 업무 개발은 **URL 여러 개**가 아니라 **`POST /업무코드/online` + serviceId** 입니다.
- **`ztcfbook-m`** 은 그걸 **쉬운 말**로, **`ztcfbook`** 은 **현장 표준 전체**입니다.
- 인프라보다 **“요청 한 번이 코드까지 어떻게 가는지”** 에 집중하면 됩니다.
